数据的训练集与测试集划分

在机器学习和数据科学中，我们经常需要将数据集分为训练集和测试集。训练集用于训练模型，而测试集用于评估模型的性能。train_test_split 是 scikit-learn 库中的一个非常有用的函数，它可以帮助我们轻松完成这个任务。

使用 train_test_split 函数

函数签名如下：

  • X 和 y：分别是特征矩阵和目标向量。
  • test_size：测试集的大小比例，通常是一个介于0和1之间的浮点数。在这个例子中，我们将30%的数据用作测试集。
  • random_state：随机数生成器的种子或 RandomState 实例，以确保每次数据划分时都能得到相同的结果。在这里，我们使用了420作为种子值。

返回值

  • X_train, X_test：划分后的特征矩阵的训练集和测试集部分。
  • y_train, y_test：划分后的目标向量的训练集和测试集部分。

PCA

sklearn.decomposition.PCA（主成分分析）在初始化时可以接受多个参数。以下是PCA类的完整签名及其参数说明：

参数解释：

  • n_components: 主成分的数量。如果未设置，则所有组件都将被保留。如果n_components在(0, 1]之间（浮点数），它将根据所解释的方差的比例来选择特征数量。例如，如果n_components设置为0.95，则选择的主成分数量将使得它们所解释的方差占原始数据总方差的95%。
  • copy: 是否在运行算法之前复制数据。如果copy=False，则原始数据将在计算过程中被修改（和可能的中心化）。通常设置为True以避免改变原始数据。
  • whiten: 当此值为True时，PCA将对组件进行白化，这意味着它将消除不同主成分之间的相关性，并使得所有主成分都具有相同的方差。这通常会使后续的学习算法效果更好。
  • svd_solver: 用于执行奇异值分解（SVD）的求解器。可接受的值有：
+ `'auto'`: 求解器将根据具体情况自动选择（基于`X.shape`和`n_components`）。
+ `'full'`: 使用标准的SVD求解器进行完全的分解。在数据量大时可能非常慢。
+ `'arpack'`: 使用截断技术的SVD求解器。对于大型数据集更快，但需要更多的内存。
+ `'randomized'`: 使用随机化SVD求解器。在大型数据集上最快，但准确性可能略有降低。
  • tol: 当svd_solver=’arpack’时，收敛的容忍度。
  • iterated_power: 当svd_solver=’randomized’时，幂迭代的次数。它可以用来提高随机向量的精确度。设置为’auto’时，将自动选择迭代次数。
  • random_state: 控制随机过程的种子。对于可重复的结果，可以设置一个固定的种子值。

PCA中的属性

PCA（主成分分析）是一种常用的数据降维方法，它通过将数据投影到一个较低维度的向量空间来减少数据的复杂性，同时尽量保留原始数据中的变异性。在PCA中，有两个重要的属性可以帮助我们理解降维后的数据：explained_variance_和explained_variance_ratio_。

1.可解释方差

explained_variance_属性表示降维后每个新特征向量（主成分）上所带的信息量大小，也即是每个主成分对应的方差值。这个值越大，说明对应的主成分包含的原始数据的信息量越多。

例如，如果我们得到的explained_variance数组为[4.22824171, 0.24267075]，这意味着第一个主成分包含了4.22824171的信息量，而第二个主成分包含了0.24267075的信息量。

2. 可解释方差贡献率

explained_variance_ratio_属性表示降维后每个新特征向量所占的信息量占原始数据总信息量的比例，也即是每个主成分的方差值占所有主成分方差值之和的比例。这个比例可以帮助我们了解每个主成分对原始数据的贡献程度。

例如，如果explained_variance_ratio_数组为[0.95, 0.05]，这意味着第一个主成分贡献了95%的信息量，而第二个主成分只贡献了5%的信息量。这也说明大部分信息都被有效地集中在了第一个特征上。

3. 查看总信息量

我们可以通过对explained_variance_ratio_求和来查看降维后保留的总信息量。如果求和结果接近1，则说明降维后的数据保留了原始数据的大部分信息。

概述

KMeans

KMeans 是 sklearn 库中用于执行 K 均值聚类算法的类。下面是该类的初始化函数签名，并附有对每个参数的详细解释。

参数说明：

  1. n_clusters: 整数，默认值=8

    要形成的簇的数量，即算法应该找到的质心数。

  2. init: {‘k-means++’, ‘random’} 或可调用对象，默认值=’k-means++’

  3. n_init: 整数，默认值=10

    使用不同的质心初始化来运行算法的次数。最终解是在这些运行中惯性最小的那个。

  4. max_iter: 整数，默认值=300

    单次运行 k-means 算法的最大迭代次数。

  5. tol: 浮点数，默认值=1e-4

    相对质心变化的收敛容忍度。当质心变化小于此值时，算法会停止。

  6. precompute_distances: {‘auto’, True, False}，默认值=’auto’

    预计算距离：

    • ‘auto’：如果 n_samples * n_clusters > 12e6，则不预计算距离。这对内存使用有益，但计算可能稍慢一些。
    • True：始终预计算距离。
    • False：从不预计算距离。

  7. verbose: 整数，默认值=0

    冗长模式。

  8. random_state: 整数、RandomState 实例或 None，默认值=None

    确定质心初始化的随机数生成。使用整数可以确保结果的可重复性。

  9. copy_x: 布尔值，默认值=True

    当我们预计算距离时或在算法中需要多次使用数据集时，如果为 True，将复制数据集；否则，只引用它。如果引用且数据集在拟合过程中被修改，可能会导致错误。

  10. n_jobs: 整数或 None，默认值=None

    用于计算的作业数。None 表示 1，除非在 joblib.parallel_backend 上下文中。-1 表示使用所有可用的处理器。

  11. algorithm: {‘auto’, ‘full’, ‘elkan’}，默认值=’auto’

    使用的 k-means 算法类型：

    • ‘full’：标准的 k-means 算法。
    • ‘elkan’：使用三角形不等式进行加速。当数据集是密集的，即当 n_samples 和 n_clusters 都很大时，通常更快。
    • ‘auto’：基于数据自动选择 ‘full’ 或 ‘elkan’。

查看总距离平方和

总距离平方和（通常称为inertia）是KMeans算法优化的目标函数，它表示所有数据点到其所属簇质心的距离平方的总和。一个较低的inertia值通常意味着聚类效果较好，因为数据点更紧密地聚集在各自的质心周围。

要查看KMeans聚类后的总距离平方和，我们可以使用inertia_属性。

请注意，inertia本身并不能直接用来比较不同数量的聚类之间的效果，因为随着聚类数量的增加，inertia通常会降低，但这并不意味着找到了真正的聚类结构。因此，在选择最佳聚类数时，通常需要结合其他指标（如轮廓系数、肘部法则等）来综合评估。

评价聚类效果

轮廓系数评价

在实际应用中，我们往往是对没有真实标签的数据进行聚类。因此，评估聚类的效果变得尤为重要。轮廓系数是一种常用的聚类算法评价指标，它能够同时衡量样本与其自身所在的簇中的其他样本的相似度以及样本与其他簇中的样本的相似度。

对于单个样本而言，轮廓系数计算为：

其中，a表示样本与其自身所在的簇中的其他样本之间的平均距离（簇内差异），b表示样本与下一个最近的簇中的所有点之间的平均距离（簇间差异）。我们希望b永远大于a，并且大得越多越好。

轮廓系数的取值范围是(-1, 1)，值越接近1表示样本与自己所在的簇中的样本很相似，并且与其他簇中的样本不相似；当样本点与簇外的样本更相似时，轮廓系数为负；当轮廓系数为0时，则代表两个簇中的样本相似度一致，两个簇本应该是一个簇。

如果一个簇中的大多数样本具有比较高的轮廓系数，则该簇会有较高的总轮廓系数。整个数据集的平均轮廓系数越高，说明聚类效果越好。如果许多样本点具有低轮廓系数甚至负值，则说明聚类效果不合适，可能需要调整聚类的超参数K的值。

在sklearn库中，我们可以使用metrics模块中的silhouette_score函数来计算轮廓系数。该函数返回的是一个数据集中所有样本的轮廓系数的均值。另外，我们还可以使用silhouette_samples函数来计算每个样本自己的轮廓系数。这些函数可以帮助我们评估聚类的效果并进行相应的调整。

文本处理

TF-IDF

fit_transform方法

fit_transform(raw_documents, y=None)方法的主要作用是学习词汇表并返回文档术语矩阵。这里的“学习词汇表”指的是，它会遍历提供的文档（raw_documents），构建一个包含所有不重复词汇的词汇表，并为每个词汇分配一个唯一的整数索引。然后，它会根据这个词汇表将文档转换为TF-IDF特征向量。

这个方法相当于先调用fit方法学习词汇表，然后再调用transform方法将文档转换为特征向量。但是，fit_transform的实现更加高效，因为它可以在一次遍历中同时完成词汇表的学习和文档的转换。

transform方法

与fit_transform不同，transform(raw_documents)方法只是将文档转换为特征向量，而不学习词汇表。这意味着，在使用transform方法之前，必须先使用fit方法或fit_transform方法来学习词汇表。否则，transform方法将无法正确地将文档转换为特征向量，因为它不知道词汇表中包含哪些词汇。

需要注意的是，在调用transform方法时，提供的文档（raw_documents）应该只包含已经在学习词汇表时出现过的词汇。如果包含了新的、未在学习词汇表时出现的词汇，那么这些词汇将被忽略，并且在转换后的特征向量中不会有任何体现。因此，在处理新的、未知的数据时，需要特别小心处理这种情况。
